# Job Application Pipeline — Master Spec
> Feed this file to a fresh Claude Code session. It contains everything needed to build the full codebase from scratch.

---

## What We Are Building

An automated job application pipeline for a Software Engineer targeting big tech (FAANG/MAANG, Stripe, Databricks, Airbnb, etc.). The system:

1. Scrapes job postings daily from multiple sources
2. Scores each posting against the candidate's profile using Claude AI
3. Generates a tailored cover letter and resume bullets for passing jobs
4. Presents a human review queue (approve / edit / skip)
5. Auto-submits approved applications via Playwright
6. Tracks all applications in a local SQLite database

**The candidate reviews every application before it is sent. Nothing is ever submitted without approval.**

---

## Tech Stack

- **Language:** Python 3.11+
- **Scraping:** `python-jobspy` for major boards, direct JSON API calls for Greenhouse/Lever
- **AI:** Anthropic Claude API (`claude-sonnet-4-5` model) for scoring + generation
- **Browser automation:** Playwright (Python) for auto-submit
- **Database:** SQLite via Python `sqlite3` stdlib
- **Scheduler:** system `cron` (instructions in README)
- **Config:** `.env` file via `python-dotenv`

## Database Schema

Single SQLite file at `data/jobs.db`. Create these tables on first run:

```sql
CREATE TABLE IF NOT EXISTS jobs (
    id TEXT PRIMARY KEY,          -- SHA256 hash of job URL
    title TEXT NOT NULL,
    company TEXT NOT NULL,
    location TEXT,
    url TEXT UNIQUE NOT NULL,
    source TEXT,                  -- 'jobspy_linkedin', 'jobspy_indeed', 'greenhouse', 'lever'
    description TEXT,
    salary_min INTEGER,
    salary_max INTEGER,
    date_posted TEXT,
    date_scraped TEXT NOT NULL,
    score INTEGER,                -- 0-100, NULL until scored
    score_reason TEXT,
    status TEXT DEFAULT 'new',    -- new | scored | queued | approved | submitted | skipped | failed
    cover_letter TEXT,
    resume_bullets TEXT,          -- JSON array of 3 tailored bullet strings
    date_submitted TEXT,
    notes TEXT
);

CREATE TABLE IF NOT EXISTS applications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    job_id TEXT REFERENCES jobs(id),
    submitted_at TEXT,
    confirmation_text TEXT,
    follow_up_date TEXT
);
```

## Layer 1 — Scrapers

### `scrapers/jobspy_scraper.py`

Use the `python-jobspy` library. Key implementation notes:

```python
from jobspy import scrape_jobs

def scrape_major_boards(preferences: dict) -> list[dict]:
    """
    Scrape LinkedIn, Indeed, Glassdoor, ZipRecruiter, Google Jobs via JobSpy.
    Returns list of normalized job dicts.
    """
    results = scrape_jobs(
        site_name=["linkedin", "indeed", "glassdoor", "zip_recruiter", "google"],
        search_term=" OR ".join(preferences["target_roles"]),
        location="United States",
        results_wanted=50,
        hours_old=24,
        linkedin_fetch_description=True,
    )
    # Convert DataFrame to list of dicts, normalize fields
    return normalize_jobspy_results(results)
```

Handle the pandas DataFrame that JobSpy returns. Map its columns to the jobs.db schema.

### `scrapers/ats_scraper.py`

No scraping needed — these are public JSON endpoints:

```python
import httpx

def fetch_greenhouse_jobs(board_token: str) -> list[dict]:
    url = f"https://api.greenhouse.io/v1/boards/{board_token}/jobs"
    resp = httpx.get(url, timeout=10)
    resp.raise_for_status()
    jobs = resp.json().get("jobs", [])
    return [normalize_greenhouse_job(j, board_token) for j in jobs]

def fetch_lever_jobs(company_slug: str) -> list[dict]:
    url = f"https://api.lever.co/v0/postings/{company_slug}?mode=json"
    resp = httpx.get(url, timeout=10)
    resp.raise_for_status()
    return [normalize_lever_job(j, company_slug) for j in resp.json()]
```

Loop over all boards in `preferences["greenhouse_boards"]` and `preferences["lever_boards"]`.

## Layer 2 — Normalizer

### `pipeline/normalizer.py`

All scrapers return different schemas. Normalize everything to match `jobs.db`:

- Generate `id` as `sha256(url)[:16]`
- Strip HTML from descriptions with `BeautifulSoup`
- Deduplicate: skip any URL already in `jobs.db`
- Set `status = 'new'`, `date_scraped = now()`

**Stage 2 — Claude semantic scoring:**

Prompt template (send as user message, system prompt sets the persona):

```
SYSTEM:
You are an expert technical recruiter. You evaluate job description fit for software engineers.
Always respond with valid JSON only. No explanation outside the JSON.

USER:
Score how well this candidate matches this job. Return JSON with exactly these fields:
{
  "score": <integer 0-100>,
  "reason": "<one sentence explaining the score>",
  "missing": ["<skill or experience gap 1>", "<gap 2>"],
  "strengths": ["<matching strength 1>", "<strength 2>"]
}

CANDIDATE PROFILE:
{profile_json}

JOB DESCRIPTION:
Title: {title}
Company: {company}
Description: {description}
```

- Use `claude-sonnet-4-5` model, `max_tokens=300`
- Parse JSON response, store `score` and `score_reason` in jobs.db
- Update `status = 'scored'`
- Jobs with `score >= SCORE_THRESHOLD` → update `status = 'queued'`

## Layer 3 — Generator

### `pipeline/generator.py`

Only called for jobs with `status = 'queued'`.

**Cover letter prompt:**

```
SYSTEM:
You are a professional cover letter writer for software engineers targeting big tech.
Write in first person. Be specific, not generic. Never use filler phrases like
"I am excited to apply" or "I am a passionate engineer."
Reference specific details from the job description. Keep it to 3 paragraphs, under 300 words.

USER:
Write a cover letter for this application.

CANDIDATE:
{profile_json}

JOB:
Title: {title}
Company: {company}
Why this role scored well: {score_reason}
Strengths matched: {strengths}
Description: {description}

Output the cover letter text only. No subject line. No "Dear Hiring Manager" header.
```

**Resume bullet tailor prompt:**

```
SYSTEM:
You are a resume writer. Rewrite the candidate's existing bullet points to better match
a specific job description. Keep the facts 100% accurate — only adjust emphasis and language.
Return a JSON array of exactly 3 bullet strings.

USER:
Rewrite the top 3 most relevant bullets from this resume to match this job description.
Use the job's language and keywords where truthful. Each bullet must start with a strong
action verb and include a measurable impact where one exists.

CANDIDATE BULLETS:
{all_bullets_json}

JOB DESCRIPTION KEYWORDS AND REQUIREMENTS:
{description}

Return JSON array only: ["bullet 1", "bullet 2", "bullet 3"]
```

- Use `claude-sonnet-4-5`, `max_tokens=600` for cover letter, `max_tokens=400` for bullets
- Store results in `jobs.db` `cover_letter` and `resume_bullets` columns
- Update `status = 'queued'` (already set, just confirm)

## Layer 4 — Review CLI

### `review/cli.py`

This is the most important UX component. The candidate interacts with this every morning.

Display format for each job:

```
────────────────────────────────────────────────────
[2/6]  Senior Software Engineer — Stripe
       San Francisco, CA · Remote OK · Full-time
       Score: 91/100  |  strong fit
       Why: distributed systems, Go, payments domain match
       Missing: Kafka experience

  COVER LETTER PREVIEW:
  ┌──────────────────────────────────────────────────┐
  │ [first 4 lines of cover letter]                  │
  └──────────────────────────────────────────────────┘

  [a] approve + submit   [e] edit   [r] read full JD   [v] view full letter   [s] skip
❯
```

Key behaviors:
- `a` → call `submitter.submit(job)`, update `status = 'submitted'`
- `e` → write cover letter to `/tmp/cover_letter.txt`, open with `$EDITOR`, read back on save, prompt "submit now? [y/n]"
- `r` → page through full job description with `less`
- `v` → page through full cover letter
- `s` → update `status = 'skipped'`, never show again
- Show `[X/Y]` counter so candidate knows how many are in queue
- After last job, print session summary: X submitted, Y skipped, Z remaining

Use `input()` for keypresses. Handle `KeyboardInterrupt` (Ctrl+C) gracefully — save progress and exit.

Build in this exact sequence. Each step is testable before moving to the next:

1. `utils/db.py` — database init + helpers (foundation for everything)
2. `utils/profile.py` — load and validate `candidate_profile.json`
3. `run.py setup` — interactive profile builder (lets candidate test immediately)

5. `scrapers/ats_scraper.py` — Greenhouse + Lever (clean JSON, easy to test)
6. `scrapers/jobspy_scraper.py` — JobSpy (more complex, test after ATS works)
7. `pipeline/normalizer.py` — merge all sources into jobs.db
8. `pipeline/generator.py` — cover letter + bullet generator
9. `review/cli.py` — review queue (test with mock data from jobs.db)
10. `pipeline/submitter.py` — Playwright auto-submit (build last)
11. `run.py` — wire all commands together
12. `README.md` — document setup and daily use

## Quality Standards

- Every function must have a docstring
- All Claude API calls must have try/except with meaningful error messages
- Database writes must use transactions
- Never crash silently — always print what went wrong and continue
- Log all submissions to `logs/submissions.log` with timestamp
- Never store passwords — only session cookies and API keys in `.env`
- All monetary values stored as integers (cents) in the database

## Notes for the Implementing Agent

- Start with `utils/db.py` and `run.py setup` — get the profile JSON working first, everything else depends on it
- The scorer is the most important component — invest time in the prompt engineering
- Test the scorer by pasting real job descriptions before building scrapers
- JobSpy returns a pandas DataFrame — handle it carefully, column names change between versions
- Greenhouse API is rate-limited at ~100 req/min — add a small sleep between board fetches
- The review CLI is the candidate's daily driver — make it clean, fast, and keyboard-friendly
- Use `rich` library for colored output in the terminal (green for approved, red for skipped, etc.)
